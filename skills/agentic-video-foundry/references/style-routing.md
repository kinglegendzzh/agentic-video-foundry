# 可扩展视觉风格路由

风格模块只定义视觉与运动语法，不接管 Foundry 的剧本、证据、音频、主时间轴和交付闸门。一个项目只能选择一个主风格；局部变化必须继承同一材质、字体、阴影和运动规则。

## 何时选择风格

1. 先诊断输入：文案/脚本、图片素材、原片视频、只有主题想法。
2. 再确定平台画幅、目标时长和真实证据计划。
3. 只询问会改变制作路径的缺失信息；已有答案不得重复追问。
4. 给出一个推荐风格、选择理由和明确排除项，经用户确认或已有明确指令后写入项目 `style-preset.json`。
5. 读取且只读取所选模块；不要把所有风格说明加载进上下文。

## 模块契约

每个 `references/styles/*.md` 必须包含：

- `id`、适用内容、输入模式和不适用情况；
- 可复制的默认 preset 路径与允许覆盖的 token；
- 材质、配色、排版、构图、真实证据处理方式；
- 逐帧运动语法、转场族、节奏和允许的例外；
- 不变量、重复角色/道具 Identity Block、逐镜进退场与连续性字段；
- 生成式关键帧/视频的全局正负提示块，以及精确文字的后期合成规则；
- 分阶段产物、确认门和硬失败项；
- 代表帧与完整成片的专项 QA。

项目 preset 至少记录：

```json
{
  "id": "style-id",
  "version": 1,
  "overrides": {},
  "evidenceTreatment": "style-specific",
  "renderer": "remotion"
}
```

允许覆盖颜色、字体、纸张纹理强度、阴影方向、切边粗糙度、定格步长和转场密度；不得覆盖平台安全区、单一 Hero、真实 Demo、确定性渲染或最终视听验收。
