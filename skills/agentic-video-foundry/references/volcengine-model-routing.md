# 火山豆包语音模型路由

本页把火山控制台里的“已开通服务”转换为视频生产决策。开通不等于默认使用；每项能力必须对应明确的输入、输出、计费单位和验收收益。

## 三档预设

| 预设 | 旁白 | 音乐 / 音效 | 对齐与复核 | 适用场景 |
|---|---|---|---|---|
| `cost` 成本最优 | 语音合成 1.0 公版音色；同稿 1–2 个短试听 | 音频生成 1.0，只生成 1 条 BGM 和 2–3 个关键 SFX | 优先使用 TTS 时间戳；仅失败时调用自动字幕打轴 | 内部样片、批量低风险内容、先验证选题 |
| `balanced` 效果/成本均衡 | 语音合成 2.0 公版音色；同稿 3 个短试听 | 音频生成 1.0，BGM 2 个候选、3–5 个关键 SFX | TTS 时间戳为主；成片仅在文本或时间戳可疑时调用录音文件识别 2.0 | 默认短视频、产品介绍、知识分享 |
| `quality` 效果优先 | 语音合成 2.0 高表现力音色；必要且已有合法样本时才评估声音复刻 2.0 | 音频生成 1.0，分镜驱动的 BGM 2–3 个候选与精细 SFX | TTS 时间戳 + 录音文件识别 2.0 转写核验；关键字幕人工校准 | 品牌主片、发布片、需长期复用的栏目 |

“效果最好”不自动等于声音复刻。若目标只是“激情但不叫卖的年轻中文男性”，优质公版 2.0 音色通常比临时、低质量样本复刻更稳。复刻只在品牌声纹本身是资产、样本合格且用户明确批准潜在槽位费用时进入候选。

准确价格以控制台当期计费页为准。成本排序依据调用数量、计费单位和用户当前免费额度做相对控制，不在 Skill 中硬编码易过期的单价。

## 能力分层

### 默认主链

- **语音合成 2.0**：默认旁白。适合自然度、韵律、气口和情绪表现要求高的社交视频；先试听公版音色。
- **音频生成 1.0**：BGM 与关键 SFX；也可通过参考音色生成短语音，但不能把创意音频生成当成所有 TTS 场景的唯一后端。
- **语音合成 1.0**：成本优先或 2.0 音色不适配时的旁白备选；选择依据必须是同稿试听，而不是音色数量。

### 质量增强支线

- **录音文件识别 2.0**：对最终旁白或母带做“实际说了什么”的独立转写核验，发现漏字、改词或异常静音；不是主字幕时间轴的默认来源。
- **自动字幕打轴**：已有准确文稿但原始人声没有可靠时间戳时，对齐文本与音轨；TTS 已返回可信字/句时间戳时不重复调用。
- **音视频字幕生成**：处理用户提供的 Demo、采访或素材原声；不用于替代本管线自己持有的 TTS alignment。
- **声音复刻 2.0**：品牌固定声线或用户本人声音；需要合格样本、speaker ID 与明确费用门槛。
- **声音复刻 1.0**：仅兼容遗留音色或已验证旧项目；新项目默认 2.0。

### 特殊节目形态

- **语音播客**：双人对谈、资料解读或栏目化播客；它会改变剧本结构和鉴权，不是单人短视频旁白的升级按钮。
- **精品长文本语音合成**：长文、有声书或批量内容；30–90 秒短视频仍走短文本分段合成。
- **机器翻译 + 同声传译 2.0**：多语分发或直播同传；普通中文成片不调用。
- **端到端实时语音、流式语音识别 1.0/2.0、一句话识别**：交互式录制、直播或语音助手；离线视频生产默认不调用。
- **语音妙记、录音文件识别 1.0**：会议/遗留兼容能力；已有 2.0 时不进入新项目首选。

## 路由规则

1. 项目默认 `balanced`；用户明确要求“最好”才升到 `quality`，明确批量/省额度才降到 `cost`。
2. 每个角色先使用同一段 8–12 秒文本试听，禁止用不同台词比较音色。
3. 旁白、BGM、SFX 可以使用不同服务，但最终只进入一个音频 manifest 和一个主时间轴。
4. 服务必须与 API Key 位于同一火山项目。`45000010` 是 Key 无效；`45000030` 是 Key 已识别但资源未授权，或开通状态尚未传播到该 Key 所属项目。
5. 新开通服务后先用最短试听验证权限。失败时保存状态码和资源 ID，不反复重试、不生成完整资产。
   批量计划必须支持按 asset ID 单条重试，避免响应兼容问题导致整批重复计费。
6. 识别模型只验证实际音频，不拿原稿“自证正确”；对齐差异进入人工复核，不静默覆盖脚本。
7. 任何档位切换都会使旧的时间轴、字幕和母带测量失效。
8. speaker ID 具有服务作用域。播客专属音色即使属于同一账户，也可能在音频生成返回 `45001115`；此时必须改用目标服务音色，或先做无 speaker 的自然语言音色试听，禁止继续猜 ID。

## 当前短视频的推荐

本项目采用 `balanced`，但把旁白质量提升到 `quality` 的试听标准：先用音频生成 1.0 做 3 个自然语言描述的年轻男声候选，选中后以试听音频作为参考保持分段声线一致；若目标服务能取得已授权的语音合成 2.0 speaker，再把它加入同稿比较。BGM 与 5 个关键 SFX 使用音频生成 1.0；不启用声音复刻；最终仅在 TTS 对齐可疑时调用录音文件识别 2.0。这样利用已开通额度，同时避免为一条 60–70 秒单人介绍片引入播客、同传和复刻的额外复杂度。
