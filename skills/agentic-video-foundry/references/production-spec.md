## 1. 输入与立项

建立制作简报并明确：平台、受众、唯一传播结论、CTA、时长范围、画幅、语言、声线角色、情绪曲线、视觉方向、证据计划、必须出现的事实、禁止项和现有素材。信息不足时可用合理默认值推进，但要将会显著改变成片的假设写入简报。

竖屏社交视频默认基线：1080×1920、30 fps、H.264、AAC 48 kHz；小红书/抖音类内容默认预留顶部 120 px、左右 72 px、底部至少 300 px，并避免在右侧约 150 px 放持续性核心文字。发布前仍应按目标平台最新界面复核。

## 2. 技术路线

### Remotion 默认适用

- React/数据驱动模板、批量变体或固定栏目；
- 严格逐帧控制、字幕与音频同步；
- 需要自动渲染、确定性复现和程序化质检；
- 场景较多、音频层复杂或后续会持续维护。

### HyperFrames 条件适用

- 已有大量可直接复用的 HTML/CSS/GSAP 组件；
- 目标主要是快速组织网页、素材和轻量 DOM 动画；
- agent-native 网页组合的收益大于迁移和对齐成本。

如果引入 HyperFrames，先写清它负责的边界、主时间轴来源、音频归属、渲染出口和像素/时序一致性验证。没有这些答案时保持 Remotion 单链。

## 3. 剧本与节奏

采用“钩子 → 价值 → 证据/覆盖 → 转折 → 核心机制 → CTA”或同等清晰的叙事弧。开头 1–3 秒必须建立冲突、收益或反常识信息。旁白负责解释关系，画面负责枚举与证明；禁止把屏幕上的全部分类逐项念一遍。

按语义段落保存脚本。每段包含 `id`、文本、场景意图与可选停顿。用真实配音时长生成连续时间轴；如果超过目标时长，优先删减重复信息。主要信息落地后保留可感知的静止或呼吸区，避免全片同速。

## 4. 视觉与动画

在编码前固定颜色、字体、字号、安全区、缓动、基础时间单位、级联节奏和峰值镜头。每帧一个 Hero，支持元素降低尺寸、对比度或运动强度。全片设置有限的运动高潮，其余镜头为高潮服务。

产品/工具介绍片默认把真实演示作为主体：用实际 UI、终端命令、仓库文件、渲染预览、对比结果或交付物证明能力。若这些证据可获得，建议整片 45%–70% 的有效观看时长由真实演示承担；动效层只做裁切、放大、框选、游标、标签、遮罩、节拍转场和结果强调。比例不是机械 KPI，但完全没有演示证据必须在简报中说明原因。

程序化视频的动画由帧值驱动；禁止依赖 CSS animation、运行时计时器、`Date.now()` 和无种子的随机数。背景纹理应低对比，不得抢过信息。字幕最多两行，优先按语义和标点切分；Codex、Claude Code、Gemini CLI、AI Skills 等拉丁短语不得从中间断开。

## 5. 配音、音乐和音效

### 配音

先把“声线”写成角色描述，例如“激情但不叫卖的年轻中文男性，像懂技术的朋友分享私货”。从语言、年龄、性别、口音、用途、情绪与录音质量筛选至少 3 个相同文案试听。API 过滤后仍要本地核对主要语言与元数据。

按段生成并保存时间戳。专有名词先试听；模型不支持上下文字段时按模型能力裁剪请求体。更换声线后重建旁白、时间轴、字幕和混音，不把旧版时间硬套到新声线。

音频供应商按 [audio-provider-routing.md](audio-provider-routing.md) 选择。使用火山时再按 [volcengine-model-routing.md](volcengine-model-routing.md) 选择 `cost`、`balanced` 或 `quality`，默认 `balanced`。新版火山短视频路径使用豆包语音/OpenSpeech 的 `X-Api-Key` 接口；不要把它写成通用“方舟模型”。复刻音色必须额外有样本和 speaker ID；长文本/播客材料若仍要求 App ID、Access Token、Resource ID，就保持独立适配，不伪装成只需 API Key。

### 音乐

提示词描述 BPM、类型、乐器、段落结构、情绪、禁用元素和明确结尾。音乐必须服务剧本起伏：钩子稀疏、主体建立律动、转折留白、结尾抬升收束。禁止使用来源不明的网络音乐。

### 音效与混音

音效只强化关键落点，不给每个元素配声。以完整编码成片为测量对象。语音内容类社交视频可将综合响度约 -16 至 -14 LUFS、真峰值不高于 -1 dBTP 作为起始验收范围；具体平台和素材优先。音乐相对旁白通常低 10–16 dB 起步，再用耳机与手机外放复核，确保“听得见但不抢词”。转折处可做短暂抽空，开头结尾做短淡入淡出。

## 6. 数据与工程结构

推荐单一事实链：

```text
script.json
  -> timestamped TTS -> voiceover + alignment
  -> timeline.json + captions.json
  -> music/SFX + secret-free audio-manifest.json
  -> frame-driven composition
  -> representative stills
  -> final MP4 + cover
```

把主题、时间轴、字幕、音频提示和场景分离。manifest 至少保存 routing profile、service、provider、model、voice ID、文本或提示词摘要、时长、文件路径与 SHA-256；不得保存 API Key。素材存在且哈希一致时默认复用，只有显式强制才重新消耗额度。

## 7. 渲染与质检

先通过单元测试、依赖与密钥扫描、确定性检查，再渲染 Hook、信息密集处、转折、运动高潮、字幕边界、CTA 等代表帧。逐张检查层级、安全区、拼写、文字截断、闪帧、合成器兼容性和品牌词。

静帧通过后渲染完整成片。检查编码、分辨率、帧率、时长、音轨采样率、综合响度、真峰值和音画落点。必须至少完整观看和收听一次；重点用手机外放复核声线是否自然、音乐是否可听、齿音/爆音/削波以及 CTA 停留是否足够。

## 8. 交付与复盘

最低交付：成片 MP4、封面、可编辑工程、最终脚本、字幕、音频素材与无密钥 manifest、发布文案、复现命令和验证摘要。保留上一版成片，直到新版通过验收。

把非显而易见且已验证的问题写入 `.learnings/`，包含现象、原因、解决方案、验证证据和提升条件。一次偶发问题不直接写入全局规则。
